{{- if eq .Values.networking.type "istio" }}
The Focusing Manager has been deployed with Istio VirtualService.

Access the service via Istio Gateway at:
  Path: {{ .Values.networking.istio.pathPrefix }}
  Gateway: {{ index .Values.networking.istio.gateways 0 }}

{{- else if eq .Values.networking.type "ingress" }}
The Focusing Manager has been deployed with Kubernetes Ingress.

Get the application URL by running:
{{- range .Values.networking.ingress.hosts }}
  {{- range .paths }}
  http{{ if $.Values.networking.ingress.tls }}s{{ end }}://{{ .host }}{{ .path }}
  {{- end }}
{{- end }}

{{- else }}
The Focusing Manager has been deployed without external networking.

Access the service within the cluster:
  export POD_NAME=$(kubectl get pods -n {{ .Release.Namespace }} -l "app.kubernetes.io/name={{ include "focusing-manager.name" . }},app.kubernetes.io/instance={{ .Release.Name }}" -o jsonpath="{.items[0].metadata.name}")
  kubectl -n {{ .Release.Namespace }} port-forward $POD_NAME 3000:3000

Then visit http://127.0.0.1:3000 in your browser.
{{- end }}

Service details:
  Name: {{ include "focusing-manager.fullname" . }}
  Namespace: {{ .Release.Namespace }}
{{- if .Values.autoscaling.enabled }}
  Autoscaling: Enabled (min: {{ .Values.autoscaling.minReplicas }}, max: {{ .Values.autoscaling.maxReplicas }})
{{- else }}
  Replicas: {{ .Values.replicaCount }}
{{- end }}

{{- if .Values.redis.enabled }}
  Redis Cache: Enabled
{{- else }}
  Redis Cache: Disabled (using memory cache)
{{- end }}

{{- if and .Values.autoscaling.enabled (not .Values.redis.enabled) }}

⚠️  WARNING: Autoscaling is enabled but Redis cache is disabled!
    With multiple replicas, in-memory cache will fragment across pods.
    For production, enable Redis cache:
      redis.enabled: true
      config.preprocessingCacheBackend: "memory<redis"
{{- end }}
